# 模块：爬虫

> **爬虫**就是"打开网页、读数据、转成结构化对象"的那部分代码。它**不管存哪**，存数据库的事交给上层。

## 一句话职责

**给一个网页地址，返回一个 JS 对象。**

例如：
- 输入：`https://bbs.example.com/article/club/12345`
- 输出：`{ url, title, posts: [{ floor, author, postedAt, contentText }, ...] }`

## 适配器模式（Adapter）

不同论坛网页结构不一样。如果把"如何解析"写死在主逻辑里，将来想支持新论坛就要改一堆地方。所以做法是：

- 主逻辑只面对一个**统一接口**（`SiteAdapter`）。
- 每个论坛实现一个**适配器**，把"我这家论坛的解析逻辑"塞进接口里。
- 启动时所有适配器通过 `side-effect import` 注册到 `registry`，主逻辑用 `siteKey`（例如 `"school-bbs"`）查表拿适配器。

```
┌──────────────────────────────┐
│ CrawlerService（业务层）       │
│   listThreads(siteKey, ...)   │
└──────────┬───────────────────┘
           │ 用 registry.getAdapter(siteKey) 拿到具体实现
           ▼
┌──────────────────────────────┐    ┌──────────────────────────────┐
│ school-bbs adapter            │    │ xx-bbs adapter (未来扩展)     │
│  listThreads / getThread / …  │    │  …                            │
└──────────────────────────────┘    └──────────────────────────────┘
```

## SiteAdapter 接口

定义在 [src/core/site-adapter.ts](../../src/core/site-adapter.ts)：

| 方法 | 必需？ | 干什么 |
|---|---|---|
| `isLoggedIn(page)` | ✓ | 检测当前页面是否已登录 |
| `login(page, credentials)` | ✓ | 用账号密码登录 |
| `listSections(page)` | 可选 | 抓"讨论区清单"（init step 1） |
| `listSectionChildren(page, key)` | 可选 | 抓某讨论区下的版面 + 子讨论区（init step 2） |
| `listPinnedThreadIds(page, key)` | 可选 | 抓某版面的置顶帖 articleId 列表（init step 3） |
| `listThreads(page, params)` | ✓ | 翻一页版面，返回帖子摘要列表（工作流程 2） |
| `getThread(page, params)` | ✓ | 抓一个帖子的所有楼层（翻页） |
| `search(page, params)` | ✓ | 站内搜索（school-bbs adapter 目前是 stub） |
| `ping(page)` | 可选 | 健康检查 |

`Page` 是 Playwright 的页面对象——可以理解为"一个被远程操控的浏览器标签页"。

## 数据契约（adapter 返回的对象长啥样）

接口里定义了一组 `Thread` / `ThreadSummary` / `Post` 等类型——所有 adapter **必须按这个形状返回**，上层才不用关心是哪家论坛。

```typescript
interface ThreadSummary {
  url: string;
  title: string;
  author?: string;
  postedAt?: string;       // ISO 8601, 例如 "2026-05-07T00:00:00+08:00"
  lastReplyAt?: string;
  replyCount?: number;
  // ... 见 site-adapter.ts
}

interface Thread {
  url: string;
  title: string;
  board?: string;
  posts: Post[];
  fetchedAt: string;
  raw?: Record<string, unknown>;  // 原始数据，留作回溯
}

interface Post {
  floor: number;           // 楼层号，0=楼主，1=沙发，2=板凳，3..N=各楼
  author: string;
  postedAt?: string;
  contentHtml: string;
  contentText: string;
  attachments?: PostAttachment[];
}
```

**`raw` 字段**：任何 adapter 觉得"上层可能用得上但不进通用契约"的数据都塞这里（例如帖子的 postId、点赞数、是否置顶）。下游消费方按需用。

## 现有适配器：`school-bbs`

实现在 [src/adapters/school-bbs/](../../src/adapters/school-bbs/)：

- `index.ts` — adapter 主体，注册自己 + 实现绝大多数方法。
- `listThreads.ts` — 版面页解析（`fetchBoardPage`） + 时间字符串解析（`dateToIso`）。
- `selectors.ts` — 这家论坛特定的复杂选择器（如果有）。

特色：
- 这是一个 SPA（单页应用），URL 用 hashbang 形式 `/#!section/xxx`。
- 帖子页面是 PIM 风格的 BBS（楼层卡片用 `<table class="article">`），需要清洗信息（发信人头、签名档、`※ 来源` 等）。
- 帖子时间格式有 7 种（`YYYY-MM-DD` / `MM-DD` / `HH:MM` / `今天` / `昨天` / `前天` / `N天前`），统一转 ISO。

## Playwright 用法约定

不直接用浏览器底层 API。一律：

1. 通过 `BrowserPool.acquire(siteKey)` 拿到 `BrowserContext`，再 `newPage()`。
2. 用 `page.goto(url, { waitUntil: 'domcontentloaded', timeout: 30000 })` 加载。
3. 用 `page.waitForSelector(...)` 或 `page.waitForFunction(...)` 等关键 DOM。
4. 用 `page.$eval` / `page.$$eval` 把抽取逻辑送进浏览器上下文执行（DOM 操作在那边更快、能用真正的 `querySelector`）。
5. **不要**写依赖具体 viewport / mouse position 的逻辑——纯 DOM 抽取，能在 headless 跑。

## 怎么添加一个新论坛 adapter

1. 在 `config/sites/` 加 yml 文件（[配置文件.md](../配置文件.md)）。
2. 新建 `src/adapters/<siteKey>/index.ts`：
   - 导入 `register` 和 `SiteAdapter` 类型。
   - 实现接口里必需的方法。
   - 最后一行 `register(adapter);`。
3. 在 [src/adapters/index.ts](../../src/adapters/index.ts) 加 `import './<siteKey>';`。
4. 跑测试，跑 `npm run init -- --site <siteKey>`。
