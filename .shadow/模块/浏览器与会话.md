# 模块：浏览器与会话

> 爬虫需要一个**真实的浏览器**来打开网页（论坛是 SPA，没浏览器解析不出来）。每次都开新浏览器太慢，所以做了池化复用；同时把"登录态"和"账号密码"分两层存好，避免每次都要重登。

## 这块负责什么

1. **浏览器池**（BrowserPool）：管 Chromium 进程的生命周期。
2. **会话状态**（storageState）：把登录后的 cookie / localStorage 存盘，下次跳过登录。
3. **登录管理**（AuthManager）：检测是否已登录，过期了自动重登。
4. **凭据存储**（credential-store）：把账号密码本地加密保存（可选）。

## BrowserPool

[src/core/browser-pool.ts](../../src/core/browser-pool.ts)。

**模型**：

- 进程里只开**一个 Chromium 实例**。
- 每个 `siteKey` 复用**一个 BrowserContext**（context 是 cookie/storage 的隔离单位）。
- `acquire(siteKey)` 拿到一个已配置好的 context，调用方自己 `newPage()`。
- `release()` 计数减一，全空闲后 `idleTimeoutMs` 内没人用就自动关浏览器。

**关键参数**（来自 `.env`，经 `parseConfig` 进来）：

```
BROWSER_HEADLESS           true / false      默认 true，开发时设 false 看着调
BROWSER_EXECUTABLE_PATH    Chrome.exe 路径   可选，默认用 Playwright 自带的 chromium
BROWSER_USER_AGENT         自定义 UA          可选
STORAGE_STATE_DIR          ./.state          保存登录状态的目录
IDLE_TIMEOUT_MS            300000 (5 分钟)   全空闲多久后关浏览器
```

## storageState（会话状态本地化）

**问题**：每次启动都要登录论坛太慢、还容易触发风控。
**做法**：登录成功后，把 Playwright context 的 `storageState()`（包括 cookies + localStorage + sessionStorage）保存成 JSON：

```
./.state/school-bbs.json    ← 文件权限 0600（Windows 上尽力）
```

下次 `acquire('school-bbs')` 时，自动 `storageState: JSON.parse(...)` 注入到新建的 context 里——直接进入已登录状态。

**清理**：会话过期（被论坛踢出）时，`wipeStorageState(siteKey)` 删掉文件 + 关掉旧 context，再让 AuthManager 重登。

## AuthManager

[src/core/auth-manager.ts](../../src/core/auth-manager.ts)。

两个公开方法：

### `ensureLoggedIn(page, adapter)`

调用 `adapter.isLoggedIn(page)` 检测。如果未登录：

1. 找一份凭据（账号密码）—— 见下一小节。
2. 调 `adapter.login(page, creds)` 走表单登录。
3. 登录成功后 `saveStorageState()` 把状态写盘。

### `detectSessionExpired(page, adapter)`

业务流程中途调用——发现 `isLoggedIn` 突然变假，返回 `SessionExpiredError`。`CrawlerService.run` 接到这个错就走"清 storageState + 重登"的外层重试。

## 凭据获取：三层兜底

`resolveCredentials(siteKey)` 按以下顺序找账号密码：

1. **环境变量**（首选）：例如 `SCHOOL_BBS_USERNAME` / `SCHOOL_BBS_PASSWORD`。变量名由 `siteKey` 大写化得到（连字符变下划线）。
2. **加密凭据文件**：上一步不存在时，读 `./.state/<siteKey>.credentials.enc`（由 `npm run login` 命令交互式录入并加密落盘）。
3. **都没有 → 抛 `MissingCredentialsError`**。

**密码安全**：

- AuthManager 拿到密码后立即 `addRedactedSecret(password)`，让 pino logger 不会把密码写进日志。
- credential-store 用 AES-256-GCM 加密，key 从 `CRED_KEY` 环境变量派生；没设则用主机名衍生一个（仅防"路过的人 `cat`"，不防针对性攻击）。

## 命令：`npm run login`

[scripts/auth/do-login.ts](../../scripts/auth/do-login.ts)：

- 交互式问账号密码。
- 用 BrowserPool + AuthManager 跑一次登录。
- 询问是否"记住密码"——记得就调 `saveCredentials(siteKey, creds)` 加密落盘。
- 退出。

之后所有任务可以无人值守跑（前提是 storageState 没过期，或者凭据已存）。

[scripts/auth/login-once.ts](../../scripts/auth/login-once.ts) 是非交互式版本，从 env 取凭据强制重登一次（用于 CI）。

## 一次完整请求的浏览器生命周期

`CrawlerService.run()` 里：

```
1. rateLimiter.acquire(siteKey)         ← 抢限速令牌（[模块/限速.md]）
2. browserPool.acquire(siteKey)         ← 拿 context（必要时启动 browser）
3. context.newPage()                     ← 开标签页
4. auth.ensureLoggedIn(page, adapter)   ← 检测/登录
5. adapter 的具体方法干活（解析）
6. finally: page.close() + context release + rate release
   （context 不立刻关，留着复用；浏览器空闲超时后才整体关掉）
```

如果第 5 步检测到 `SessionExpiredError`，外层会 `wipeStorageState` 然后从第 2 步重来一次。
