# 工作流程 2：爬取指定版面的帖子

> 初始化只抓了"骨架 + 置顶帖"。**普通帖子按需抓**——用户/agent 说"我要北邮人在上海这个版面的最新帖子"，本工作流才动手。

## 一句话流程

```
按版面名找到版面 → 一页一页翻 → 解析每页帖子行 → 落库
```

可选是否同时把每个帖子的**回复楼层**也拉下来（默认只抓帖子标题/作者/时间这些列表层信息）。

## 用户场景

最典型的调用方式：

```bash
npm run crawl:board -- --board "北邮人在上海"
npm run crawl:board -- --board "北邮人在上海" --mode pages --pages 5
npm run crawl:board -- --board "北邮人在上海" --include-replies
```

或者从代码里调：

```typescript
const result = await crawlerService.listThreadsByName({
  siteKey: 'school-bbs',
  boardName: '北邮人在上海',
  mode: 'incremental',  // 或 'pages'
});
```

返回**这次爬到的帖子列表** + **下次该从第几页开始**（`nextCursor`）+ **爬取进度状态**（`state`）。

## 两种模式

### 模式 A：增量（`mode: 'incremental'`，默认）

适合定期刷新。

- 从第 1 页开始翻。
- 每翻一页，看每条帖子的 `posted_at`。
- 一旦发现某条非置顶帖的 `posted_at <= 上次记录的最新发帖时间`（这条记录叫 **watermark**），就停。
- 置顶帖始终往下走——它们不按时间排，不能作为停止信号。

**好处**：第一次跑会爬到底（watermark 为空），之后每次只抓**真正新增**的帖子，省时间省带宽。

**安全阀**：硬上限 `INCREMENTAL_PAGE_CAP = 50` 页。极端情况（watermark 错了、论坛重置）也不会无限翻。

### 模式 B：固定页数（`mode: 'pages'`）

适合"我就是想看前 N 页"或者"分段补抓"。

- 从 `cursor.startPage`（默认 1）开始。
- 翻 `pages` 页（默认 3）。
- 返回 `nextCursor.startPage = 当前最大页 + 1`，方便外部"继续翻下一段"。

## 翻页机制

具体在 `src/adapters/school-bbs/listThreads.ts:fetchBoardPage()` 实现：

- URL 模板：`/#!board/{boardKey}?p={pageNum}`（第 1 页省略 `?p=`）。
- 等待页面 hash 变为 `#!board/...` **且** DOM 出现 `a[href^="/article/"]`，然后读取 `<tbody> <tr>`。
- 每行解析出：`articleId`、标题、作者、发帖时间、最后回复时间、回复数、是否置顶。
- 从 `<ol class="page-main">` 读总页数 `totalPages`。
- 翻页间隔由 `crawl.pageTurnIntervalMs` 控制（默认 100ms，外加限速器兜底）。

## 时间字段处理

论坛上的日期写法五花八门（`2026-05-07`、`05-07`、`14:30`、`今天`、`昨天`、`3天前`），全部由 `dateToIso()` 转成 ISO 8601 字符串（CST 时区，UTC+8）。详见 `src/adapters/school-bbs/listThreads.ts:dateToIso`。

转换不了的（罕见的奇怪格式）返回 `undefined`，调用方按"没有时间信号"处理（增量爬取不会因为这条帖子停下）。

## 进度状态（watermark）

每个版面在 `board_crawl_state` 表里有一行记录：

| 字段 | 含义 |
|---|---|
| `deepest_page_crawled` | 历史上爬到过的最深页码（取 max） |
| `latest_thread_posted_at` | 历史上看到过的最新发帖时间（取 max，作为增量 watermark） |
| `last_crawled_at` | 最近一次爬取完成时间 |

每次爬取结束，新观察到的最大 `posted_at` 会更新 watermark；新到的最深页码会更新 `deepest_page_crawled`（取 max，不回退）。

## 是否抓回复？

工作流默认**只抓帖子列表层**（标题/作者/时间/回复数），写进 `threads` 表，**不进 `posts` 表**。

如果调用方明确需要回复内容（`--include-replies` 或 `includeReplies: true`），对**这次列表里返回的每一个帖子**额外调一次 `adapter.getThread(url)`，把所有楼层 upsert 进 `posts`。

⚠️ **注意**：开 `include-replies` 会让请求数膨胀 N 倍（N = 这次列表里帖子数 × 帖子页数）。**不要常规开**，按需用。

## 错误处理

继承 [模块/重试与错误.md](../模块/重试与错误.md) 的两层重试机制：

- **NavigationTimeoutError**：单页超时，自动重试两次（500ms / 1500ms 退避）。
- **RateLimitedError**：被论坛限速，等 30 秒重试一次。
- **SessionExpiredError**：会话过期，清掉 storageState 重登一次。
- **BoardNotFoundError**：版面名找不到 → 直接抛出，不重试（用户/agent 应该用 `listBoards()` 看看正确名字）。

## 命令行入口

```bash
# 单版面增量
npm run crawl:board -- --board "北邮人在上海"

# 单版面前 5 页
npm run crawl:board -- --board "北邮人在上海" --mode pages --pages 5

# 单版面 + 抓所有回复
npm run crawl:board -- --board "北邮人在上海" --include-replies

# 整个讨论区下的所有版面（仍然按上面规则一个一个跑）
npm run crawl:section -- --section "校园生活"

# 跳过已存在帖子的快速增量
npm run crawl:board-skip -- --board "北邮人在上海"
```
