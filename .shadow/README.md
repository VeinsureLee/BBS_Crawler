# BBS_Crawler 项目设计文档

> 这个文件夹放的是**给人看的设计稿**，不是给机器读的配置。无论你是开发者还是没写过代码的人，都应该能从这里看懂项目在做什么、为什么这样做。

## 这个项目是什么

一句话：**爬学校论坛的数据，存到本地数据库，并对外暴露一些"查/改"接口。**

把它当成一个**论坛数据搬运工**就行了：

- 它定期登录论坛，把"讨论区 → 版面 → 帖子"这些结构和内容抓回来；
- 抓回来的东西按规则塞进 SQLite 数据库（一种本地文件数据库）；
- 别的程序需要查论坛数据时，调本项目的接口拿。

## 这个项目**不是**什么

很重要的边界：

- **不是 MCP 服务**。MCP（让 AI 调用工具的协议）是另一个项目要做的事，那边会基于本项目的数据库做智能检索、整理、问答。本项目只负责"拿到原料 + 存好原料"。
- **不是搜索引擎**。代码里有一个 `searchCache`（用 SQL `LIKE` 做关键词模糊匹配），那是历史遗留，不主推使用。
- **不是抓取所有论坛的通用框架**，虽然代码做了适配器抽象，但目前只实现了 `school-bbs` 一个适配器。

## 两类核心工作流

只有两件事要做，看[工作流程/](工作流程/)：

1. **初始化（`npm run init`）**：把论坛的骨架抓下来。
   - 抓"讨论区"清单（例如"本站十大""社团"）
   - 抓每个讨论区下的"版面"清单（二级、三级讨论区也一起递归）
   - 抓每个版面的"置顶帖"内容

2. **爬取指定版面的帖子（`npm run crawl:board`）**：用户点名要"北邮人在上海"这个版面，就把它的帖子列表（可含回复）抓下来。支持翻页、支持增量（只抓新内容）。

## 技术栈一句话

- **语言/运行时**：TypeScript + Node.js 20+
- **网页抓取**：[Playwright](https://playwright.dev)（一个能远程操控真实浏览器的库）
- **存储**：[SQLite](https://sqlite.org)（单文件数据库，没有服务进程）+ [better-sqlite3](https://github.com/WiseLibs/better-sqlite3) 驱动
- **配置校验**：[zod](https://zod.dev)（写 schema、运行时检查）
- **日志**：[pino](https://getpino.io)（结构化 JSON 日志）

## 阅读建议

- 第一次看：按目录顺序读 `README → 数据库 → 配置文件 → 工作流程/01 → 工作流程/02`，能形成完整心智模型。
- 想知道"某个 .ts 文件归哪管"：去对应的"模块/"文件里翻"相关代码"那节，是反向索引。
- 想动手改代码：先读相关模块的设计文档，确认你想改的是设计层面还是实现细节。设计层面的改动需要回过来更新本文档。

## 编写约定

写这些文档时遵守：

1. **先讲人话**，再上技术细节。每个文件第一段必须是"对完全不懂的人也能读懂"的概述。
2. **不为了通俗牺牲准确性**。如果"通俗版"会误导读者，那就老老实实写技术细节，并在旁边加一句"如果看不懂，记住一句话就行：……"
3. **每个模块/流程文档末尾必须有"相关代码"一节**，列出对应的 `.ts` 文件路径和它的职责。这是设计 ↔ 实现的双向锚点。
